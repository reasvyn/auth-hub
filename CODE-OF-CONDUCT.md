# Code of Conduct

## Our Commitment

We want Auth-TS to be a respectful, professional, and welcoming project for everyone who participates. Contributors, maintainers, and community members are expected to collaborate in good faith and help keep discussions constructive.

## Expected Behavior

Examples of behavior that support a healthy community include:

- Being respectful in language, tone, and feedback
- Assuming good intent while discussing technical decisions
- Giving actionable, constructive review comments
- Accepting feedback gracefully
- Focusing on what is best for the project and its users
- Helping keep security-related conversations responsible and private when needed

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, threats, or personal attacks
- Discriminatory, abusive, or insulting language
- Trolling, baiting, or deliberately hostile interactions
- Publishing private information without permission
- Disruptive conduct that repeatedly derails collaboration
- Publicly disclosing unpatched security issues irresponsibly

## Scope

This Code of Conduct applies to:

- Issues, pull requests, discussions, and code review
- Project documentation and examples
- Community spaces and direct interactions related to this repository

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, pull requests, or other contributions that violate this Code of Conduct. They may also restrict participation when necessary to protect the project and its contributors.

## Project Standards

In addition to respectful conduct, contributors are expected to:

- Follow the repository contribution workflow
- Keep changes focused and technically responsible
- Update documentation when public behavior changes
- Treat authentication and security-sensitive areas with extra care
